Obviousness

  • Use more words in type names and function names. Write more comments.
  • Consistency of terms. Do your terms match the customer's terms?
  • Get feedback on your interface. Does the design or the interface seem obvious to other.
  • Let your documentation guide the less obvious pieces.

Emergent Design

  • Its really hard to know what you need to create up front.
  • Iterate vertically. Make them regret ever coming up with the MVP acronym.
  • Try writing small design documents.
  • Look into Domain Driven Design

Helpful Constraints

  • If a value or dependency is required make it hard to NOT provide those values.
  • Try to create constraints that can be enforced by the compiler or the linter.
  • There is a reason constructor injection exists.

Abstractions

"You should de‑duplicate twice as often as you do now; and only half as often as you think you should."

- Unknown

  • Create abstractions based on like concepts, not on like data structures.

  • Good abstractions often evolve out of a lot of trial and error.

    At least thats been my experience.
  • Experiment with abstractions, but be ready to throw it all away and start again.
  • Beware the Inner Platform Effect
  • Are you trying to write The Next Great American API? That's gonna take you a while.
  • Go for obvious, not clever.

Lets get a little more practical

Think about your API's entry point.

  • Having a clear entry point that fans out to the rest of the API can help make it more discoverable.
  • If applicable, think about how an IDE's autocomplete might help or hurt how discoverable the API is.
  • For REST APIs look at tools like Swagger.
  • NUnit vs. Shouldly entry point.

Events and Messaging

  • Be wary of things like:
    • In process messaging.
    • The MediatR library. (Not to be confused with the mediator pattern)
    • Domain events. (Different than integration events)
  • These often are tight coupling in disguise.
  • Experiment with composition first.
  • Integration events (Event busses, etc.) can be an awesome way to decouple things.
